= Modificar uma assinatura
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-put]#PUT# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/subscriptions/{subscription_id}`#

Substitui todo o recurso armazenado pelo corpo da solicitação JSON fornecida, preservando os valores-chave que não podem ser modificados pelo usuário.

*Efeitos Colaterais*

* Caso o usuário cancele a assinatura, o endpoint PUT deve ser chamado com "status": "inativo".
* Quando o status é definido como "inativo", as cobranças pendentes desta assinatura são enviadas para Zuora e a assinatura Zuora será marcada como "inativa" de alguma forma.




== Parâmetros

[cols="5*"]
|===
| Nome | Tipo | Em | Obrigatório | Descrição 


| id_da_conta | string | caminho | Verdadeiro  a| 
ID do recurso de conta que o contém

* exemplo: {{.Account}}




| id_da_assinatura | string | caminho | Verdadeiro  a| 
ID da coleção de assinaturas a ser listada

|===


== Corpo da solicitação

Substitui todo o recurso armazenado pelo corpo da solicitação JSON fornecida, preservando os valores-chave que não podem ser modificados pelo usuário.

*Efeitos Colaterais*

* Caso o usuário cancele a assinatura, o endpoint PUT deve ser chamado com "status": "inativo".
* Quando o status é definido como "inativo", as cobranças pendentes desta assinatura são enviadas para Zuora e a assinatura Zuora será marcada como "inativa" de alguma forma.


[cols="4*"]
|===
| Nome | Tipo | Obrigatório | Descrição 


| tipo | string | Verdadeiro  a| 
Tipo de mídia do recurso. Os valores definidos são:

* "application/astra-subscription"




| versão | string | Verdadeiro  a| 
Versão do recurso. Os valores definidos são:

* "1,0"
* "1,1"
* "1,2"




| id | string | Falso  a| 
Identificador globalmente único do recurso. Os valores definidos são:

* Em conformidade com o esquema UUIDv4




| ID do perfil do cliente | string | Falso  a| 
String JSON contendo o ID do perfil do cliente de  http://authorize.net["Authorize.net"], que foi invocada pela interface do usuário, não está definida para o período de carência. Retorna uma string vazia durante a operação de leitura, caso não esteja definida. Os valores definidos são:

* Comprimento mínimo: 0
* Comprimento máximo: 63




| paymentFirstName | string | Falso  a| 
String JSON contendo um nome completo especificado pelo usuário, usado para faturamento de pagamentos. Especificado apenas quando a assinatura está sendo renovada. Os valores definidos são:

* Comprimento mínimo: 1
* Comprimento máximo: 63




| paymentLastName | string | Falso  a| 
String JSON contendo o sobrenome completo especificado pelo usuário, utilizado para faturamento da assinatura. Especificado apenas quando a assinatura está sendo renovada. Os valores definidos são:

* Comprimento mínimo: 1
* Comprimento máximo: 63




| paymentAddress | link:#paymentAddress["paymentAddress"] | Falso  a| 
Objeto JSON contendo uma coleção de strings JSON que especificam um endereço de cobrança. Especificado apenas quando a assinatura está sendo renovada. Se esta chave existir, todas as seis strings JSON contidas estarão presentes. Se nada for fornecido pelo usuário, a string estará vazia.



| ID do perfil de pagamento | string | Falso  a| 
String JSON contendo o ID do perfil de pagamento  http://Authorize.net["Authorize.net"] que foi invocado pela interface do usuário, não definido para o período de avaliação. Retorna uma string vazia durante a operação de leitura se não estiver definido. Os valores definidos são:

* Comprimento mínimo: 0
* Comprimento máximo: 63




| paymentExpiry | string | Falso  a| 
String JSON contendo um timestamp indicando quando o método de pagamento expira. Este valor é obtido do authorize.net(http://authorize.net[] e definido pela interface do usuário. Os valores definidos são:

* Em conformidade com o padrão de data e hora ISO-8601 Este campo não é retornado durante o período de avaliação.




| purchaseOrderNumber | string | Falso  a| 
String JSON contendo o número do pedido de compra para o método de pagamento. Os valores definidos são:

* Comprimento mínimo: 1
* Comprimento máximo: 31




| marketplace | string | Falso  a| 
String JSON que indica o marketplace para uma conta onde os pagamentos serão registrados. Valores definidos:

* "netapp"
* "azure"
* "aws"
* "gcp"




| licenseSN | string | Falso  a| 
String JSON contendo o número de série da licença para o método de pagamento. Os valores definidos são:

* Comprimento mínimo: 1
* Comprimento máximo: 31




| termos | string | Falso  a| 
String JSON que indica o nível atual da conta. Os valores definidos são:

* "teste"
* "pago"




| status | string | Falso  a| 
String JSON que indica o status atual da assinatura. Os valores definidos são:

* "ativo"
* "inativo"




| appLimit | número | Falso  a| 
Número JSON que indica o limite de aplicativos para esta assinatura. O valor -1 indica que não há limite.



| namespaceLimit | número | Falso  a| 
Número JSON que indica o limite de namespace para esta assinatura. O valor -1 indica que não há limite.



| subscriptionPeriod | número | Falso  a| 
Número JSON que indica o período de assinatura. O valor -1 indica que não há limite. Unidade: dias



| gracePeriod | número | Falso  a| 
Número JSON que indica o período de carência após o término da assinatura de avaliação ou o cancelamento de uma assinatura paga. Unidade: dias



| reminderBeforePeriod | número | Falso  a| 
Número JSON que indica a quantidade de dias antes do término da assinatura de avaliação, quando as notificações ao cliente serão iniciadas. O valor é expresso em dias, sendo -1 indicativo de que não se aplica.



| onboardStatus | string | Falso  a| 
String JSON que indica o status atual do processo de integração do cliente. Os valores válidos são "não iniciado", "em andamento", "sucesso", "falha". Os valores definidos são:

* "não iniciado"
* "em andamento"
* "sucesso"
* "falhou"




| costPerAppUnit | número | Falso  a| 
Número JSON que indica o custo por unidade do aplicativo. Para uma versão de avaliação, o valor é 0. Unidade: dólares americanos



| costPerNamespaceUnit | número | Falso  a| 
Número JSON que indica o custo por unidade de namespace. Para uma versão de avaliação, o valor é 0. Unidade: dólares americanos



| metadados | link:#type_astra_metadata_update["type_astra_metadados_update"] | Falso  a| 
Metadados especificados pelo cliente e pelo serviço associados ao recurso. Os valores definidos são:

* Está em conformidade com o esquema de metadados Astra. Se não for especificado na criação, um objeto de metadados será criado sem rótulos. Se não for especificado na atualização, os rótulos do objeto de metadados, creationTimestamp e createdBy, serão preservados sem modificação.
