= Criar um novo bucket
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-post]#POST# [.api-doc-code-block]#`/accounts/{account_id}/topology/v1/buckets`#

Indica os valores desejados para o recurso da API Bucket a ser criado.



== Parâmetros

[cols="5*"]
|===
| Nome | Tipo | Em | Obrigatório | Descrição 


| id_da_conta | string | caminho | Verdadeiro  a| 
ID do recurso de conta que o contém

* exemplo: {{.Account}}


|===


== Corpo da solicitação

Indica os valores desejados para o recurso da API Bucket a ser criado.

[cols="4*"]
|===
| Nome | Tipo | Obrigatório | Descrição 


| tipo | string | Verdadeiro  a| 
Tipo de mídia do recurso. Os valores definidos são:

* "application/astra-bucket"




| versão | string | Verdadeiro  a| 
Versão do recurso. Os valores definidos são:

* "1,0"
* "1,1"
* "1,2"




| nome | string | Falso  a| 
String JSON contendo um nome atribuído pelo usuário para o bucket. Os valores definidos são:

* Comprimento mínimo: 1
* Comprimento máximo: 256




| ID da credencial | string | Verdadeiro  a| 
String JSON contendo o identificador das credenciais usadas para acessar o bucket. Os valores definidos são:

* Conforme o esquema de identificador Astra




| provedor | string | Verdadeiro  a| 
String JSON que indica o protocolo do bucket que pode ser usado para acessar o bucket. Os valores definidos são:

* "ontap-s3"
* "storagegrid-s3"
* "generic-s3"
* "gcp"
* "azure"
* "aws"




| bucketParameters | link:#bucketParameters["bucketParameters"] | Verdadeiro  a| 
Objeto JSON que fornece informações específicas do provedor do bucket, necessárias para acessar o bucket. Os valores específicos neste bucket variam de acordo com o provedor.



| metadados | link:#type_astra_metadata_update["type_astra_metadados_update"] | Falso  a| 
Metadados especificados pelo cliente e pelo serviço associados ao recurso. Os valores definidos são:

* Está em conformidade com o esquema de metadados Astra. Se não for especificado na criação, um objeto de metadados será criado sem rótulos. Se não for especificado na atualização, os rótulos do objeto de metadados, creationTimestamp e createdBy, serão preservados sem modificação.


|===
.Exemplo de solicitação
[%collapsible]
====
[source, json, subs="+macros"]
----
{
  "type": "application/astra-bucket",
  "version": "1.2",
  "name": "Test Bucket",
  "credentialID": "d5b3854c-38de-42c6-9269-b5c052aba76f",
  "provider": "gcp",
  "bucketParameters": {
    "gcp": {
      "bucketName": "bucketName"
    }
  }
}
----
====

Versão do recurso. Os valores definidos são:

Identificador globalmente único do recurso. Os valores definidos são:

String JSON contendo um nome atribuído pelo usuário para o bucket. Os valores definidos são:

* Comprimento mínimo: 1
* Comprimento máximo: 256




| ID da credencial | string | Verdadeiro  a| 
String JSON contendo o identificador das credenciais usadas para acessar o bucket. Os valores definidos são:

* Conforme o esquema de identificador Astra




| estado | string | Verdadeiro  a| 
String JSON contendo um valor que indica o estado operacional do bucket. Os valores definidos são:

* "Pendente" - O bucket está programado para descoberta
* "disponível" - O bucket está disponível para uso
* "failed" - O bucket está em estado de falha permanente
* "removido" - O bucket não está acessível no momento
* "desconhecido" - O estado do bucket não pode ser determinado




| stateDetails | array[link:#type_astra_stateDetail["type_astra_stateDetail"]] | Verdadeiro  a| 
Matriz JSON de objetos JSON que lista detalhes adicionais sobre o motivo da transição do "state" para "removed" ou "failed". Se não houver detalhes disponíveis, essa matriz estará vazia.

* Itens mínimos: 0




| retentionTime | número | Falso  a| 
Número de segundos que representa o tempo durante o qual um arquivo neste bucket será retido. Os valores definidos são:

* -1 (Balde não suportado)
* 0 (sem política de retenção conhecida)
* 
+

____
0 (Known política de retenção)

____

* "ontap-s3"

* "generic-s3"
* "gcp"
* "azure"
* "aws"




| bucketParameters | link:#bucketParameters["bucketParameters"] | Verdadeiro  a| 
Objeto JSON que fornece informações específicas do provedor do bucket, necessárias para acessar o bucket. Os valores específicos neste bucket variam de acordo com o provedor.
